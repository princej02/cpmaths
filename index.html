Number Theory

Divisibility, primes, GCD — the bedrock of ~30% of CF problems.

● Starting Easy — GCD & Primes

GCD — Greatest Common Divisor Easy

You know this from A-Level. But you need the Euclidean Algorithm — it's O(log min(a,b)) and the most common maths function in competitive programming.

Euclidean Algorithm gcd(a, b) = gcd(b, a mod b)
gcd(a, 0) = a
Example: gcd(48, 18) → gcd(18, 12) → gcd(12, 6) → gcd(6, 0) = 6
LCM from GCD lcm(a, b) = (a × b) / gcd(a, b)
⚠️ Compute as a / gcd(a,b) × b to avoid overflow before dividing!
# Python — built in! But know the manual version
from math import gcd, lcm

# Manual Euclidean (know this for Go/C#)
def gcd(a, b):
    while b:
        a, b = b, a % b
    return a

// Go
func gcd(a, b int) int {
    for b != 0 { a, b = b, a%b }
    return a
}

# Sieve of Eratosthenes — finds all primes up to N
def sieve(n):
    is_prime = [True] * (n + 1)
    is_prime[0] = is_prime[1] = False
    for i in range(2, int(n**0.5) + 1):
        if is_prime[i]:
            for j in range(i*i, n+1, i):
                is_prime[j] = False
    return [i for i, p in enumerate(is_prime) if p]

# Single prime check — O(√n)
def is_prime(n):
    if n < 2: return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0: return False
    return True

Prime Factorisation Easy–Medium

Fundamental Theorem of Arithmetic Every integer n > 1 has a unique prime factorisation.
360 = 2³ × 3² × 5
Number of divisors of n = (3+1)(2+1)(1+1) = 24. Formula: ∏(eₖ+1) for each prime power pₖ^eₖ
Divisor Count Trick: If the number of divisors is odd, n is a perfect square. This comes up often in "count arrays with property X" problems.

Combinatorics

Counting without listing — the art of "how many ways".

● Refreshing A-Level — Permutations & Combinations

You covered nCr and nPr at A-Level (Statistics). The competitive version needs you to compute them fast under modulo and recognise which formula applies in disguised problems.

The Core Formulas Easy

Permutations — order matters P(n, r) = n! / (n−r)!
Arrange r items from n. e.g. 3-digit codes from 9 digits.
Combinations — order doesn't matter C(n, r) = n! / (r! × (n−r)!)
Choose r items from n. e.g. lottery, subsets.

Recognising Combinatorics in Disguise Medium

Grid Paths

How many ways to go from (0,0) to (m,n) moving only right or down?

C(m+n, m) = C(m+n, n)

You make m+n moves total, choose which m are "right".

Stars & Bars

Distribute n identical objects into k distinct bins (bins can be empty):

C(n+k−1, k−1)

e.g. How many ways to roll 3 dice summing to 10?

Derangements

Permutations where no element appears in its original position:

D(n) = n! × Σ(−1)ᵢ/i! for i=0..n

D(n) ≈ n!/e. D(1)=0, D(2)=1, D(3)=2, D(4)=9

Catalan Numbers

Count balanced parentheses, BSTs with n nodes, triangulations:

Cₙ = C(2n, n) / (n+1)

C₀=1, C₁=1, C₂=2, C₃=5, C₄=14, C₅=42

Inclusion-Exclusion Principle Medium

Count elements in at least one set by adding singles, subtracting pairwise intersections, adding triple intersections...

Inclusion-Exclusion for 3 sets |A ∪ B ∪ C| = |A| + |B| + |C|
              − |A∩B| − |A∩C| − |B∩C|
              + |A∩B∩C|
Alternate signs: + for odd-size subsets, − for even-size subsets.
Classic Problem: How many integers from 1 to 100 are divisible by 2, 3, or 5?
|A|=50, |B|=33, |C|=20 → −|AB|=16, −|AC|=10, −|BC|=6 → +|ABC|=3
Answer = 50+33+20−16−10−6+3 = 74

Pascal's Triangle & Binomial Theorem Easy–Medium

Pascal's Triangle — C(n,r) = C(n-1,r-1) + C(n-1,r)

Row sum = 2ⁿ. Sum of alternating row signs = 0. Diagonal sums = Fibonacci!
Binomial Theorem (you know this from A-Level C4/FP1) (a + b)ⁿ = Σ C(n,k) × aⁿ⁻ᵏ × bᵏ for k=0 to n
Coefficient of xᵏ in (1+x)ⁿ is C(n,k). Critical for generating functions.

Precomputing Factorials (The Speed Trick) Medium

If you need C(n,r) for many queries, precompute factorials and their modular inverses. Then each query is O(1).

# Precompute C(n,r) mod p for multiple queries
MOD = 10**9 + 7
N = 10**6

fact = [1] * (N + 1)
for i in range(1, N + 1):
    fact[i] = fact[i-1] * i % MOD

inv_fact = [1] * (N + 1)
inv_fact[N] = pow(fact[N], MOD-2, MOD)  # Fermat's little theorem
for i in range(N-1, -1, -1):
    inv_fact[i] = inv_fact[i+1] * (i+1) % MOD

def comb(n, r):
    if r < 0 or r > n: return 0
    return fact[n] * inv_fact[r] % MOD * inv_fact[n-r] % MOD

Modular Arithmetic

The clock arithmetic that makes huge numbers tractable — and the source of 80% of "why is my answer wrong" bugs.

⚠️ Why mod (10⁹+7)? Problems with huge answers ask for the result modulo a prime. 10⁹+7 is prime, fits in 32-bit int, and two values multiplied fit in 64-bit. It's the standard. Know it. Love it.

The Rules (Refresh from A-Level) Easy

Addition & Subtraction (a + b) mod m = ((a mod m) + (b mod m)) mod m
Safe to take mod at each step. Never let numbers grow huge.
Multiplication (a × b) mod m = ((a mod m) × (b mod m)) mod m
⚠️ Use 64-bit int or Python handles it automatically.
Division — NOT directly possible! (a / b) mod m ≠ (a mod m) / (b mod m) ← WRONG!
You need the modular inverse: (a / b) mod m = (a × b⁻¹) mod m

Fast Exponentiation (Binary Exponentiation) Easy–Medium

Computing an mod m naively is O(n). Too slow if n = 1018. Use repeated squaring for O(log n).

2¹³ via repeated squaring — 13 in binary is 1101
2¹ = 2
2² = 4
2⁴ = 16
2⁸ = 256
2¹³ = 2⁸ × 2⁴ × 2¹ = 256 × 16 × 2 = 8192
Only 4 multiplications instead of 13! 
# Python has built-in — USE IT
result = pow(base, exp, mod)  # O(log exp) automatically

// Go — implement manually
func powMod(base, exp, mod int64) int64 {
    result := int64(1)
    base %= mod
    for exp > 0 {
        if exp%2 == 1 { result = result * base % mod }
        exp /= 2
        base = base * base % mod
    }
    return result
}

Modular Inverse & Fermat's Little Theorem Hard

Fermat's Little Theorem — p must be prime! If p is prime and gcd(a, p) = 1, then: aᵖ⁻¹ ≡ 1 (mod p)
Therefore: a⁻¹ ≡ aᵖ⁻² (mod p)
Modular inverse of a = pow(a, MOD-2, MOD). This is why MOD = 10⁹+7 must be prime.
# Modular inverse — enables division under modulo
MOD = 10**9 + 7

def mod_inv(a):
    return pow(a, MOD - 2, MOD)

# Division under mod: a/b = a * mod_inv(b)
result = a * mod_inv(b) % MOD
Real Example: C(1000, 500) mod (10⁹+7).
= fact[1000] * inv_fact[500] * inv_fact[500] mod MOD.
Each inv_fact[k] = pow(fact[k], MOD-2, MOD). Precompute once → O(1) per query.

Common Modular Mistakes to Avoid

❌ Forgetting negative mod

In Python -1 % MOD = MOD-1 (correct!). In Go/C#, -1 % MOD = -1 (wrong!). Fix: ((x % MOD) + MOD) % MOD

❌ Overflow before mod

In C# with int32: a*b overflows before %MOD. Use long / int64 for intermediate products.

⚠️ Division under non-prime mod

Fermat's Little Theorem only works when mod is prime. For composite mod, use Extended Euclidean.

⚠️ Not taking mod at every step

In DP: dp[i] = (dp[i-1] + dp[i-2]) % MOD at each step, not just at the end.

Bitmask DP — The Power Move Hard

Use an integer as a set. Bit k = 1 means element k is "in the set". Enables DP over all 2ⁿ subsets for small n (typically n ≤ 20).

Bitmask representing subsets of {0,1,2,3}
0000 = {}
0001 = {0}
0010 = {1}
0011 = {0,1}
0100 = {2}
0101 = {0,2}
1111 = {0,1,2,3}
Iterate all subsets: for mask in range(1 << n)
Add element i to mask: mask | (1 << i)
Remove element i: mask & ~(1 << i)
Is i in mask? (mask >> i) & 1
Classic Bitmask DP: Travelling Salesman Problem (TSP). State = dp[mask][node] = min cost to visit nodes in mask, ending at node. n=20 → 2²⁰ × 20 ≈ 20 million states. Feasible!

Vectors & the Cross Product Medium

In A-Level you did dot products. Competitive geometry lives and dies by the cross product — specifically the 2D version which gives a signed area.

2D Cross Product of vectors OA and OB cross(OA, OB) = OA.x × OB.y − OA.y × OB.x
> 0: B is to the LEFT of OA (counterclockwise)
< 0: B is to the RIGHT of OA (clockwise)
= 0: A, O, B are collinear

Polygon Area (Shoelace Formula) Medium

Shoelace / Gauss's Area Formula Area = ½ |Σ (xᵢ × yᵢ₊₁ − xᵢ₊₁ × yᵢ)|
Sum over all edges (wrapping last point to first). Works for any non-self-intersecting polygon.
# Polygon area — O(n)
def polygon_area(points):
    n = len(points)
    area = 0
    for i in range(n):
        j = (i + 1) % n
        area += points[i][0] * points[j][1]
        area -= points[j][0] * points[i][1]
    return abs(area) / 2

Probability & Expected Value

From A-Level S1/S2 to DP on probability states.

🇬🇧 A-Level Base: You covered probability from S1 — tree diagrams, Bayes, independence, binomial distribution. This tab builds on that for competitive maths.

Expected Value — The Core Tool Easy

Definition — same as A-Level S1 E[X] = Σ xᵢ × P(X = xᵢ)
Linearity of expectation: E[X + Y] = E[X] + E[Y] — even if X and Y are NOT independent! This is the key theorem.
Linearity of Expectation in Action: Expected number of fixed points in a random permutation of n elements?
Let Xᵢ = 1 if element i is in position i. E[Xᵢ] = 1/n.
E[total] = E[X₁ + X₂ + ... + Xₙ] = n × (1/n) = 1. Always 1, regardless of n!

Geometric Distribution & Waiting Times Medium

Expected trials until first success (probability p per trial) E[trials] = 1/p
Coupon Collector: to collect all n coupons (uniform random), expected draws = n × Hₙ = n(1 + 1/2 + 1/3 + ... + 1/n) ≈ n ln n

Probability DP Hard

Many problems ask for expected value of a process. Define dp[state] = expected value from that state. Transition using probability-weighted sums.

Classic: Expected steps to reach state N from 0 (dice roll)
E[0] = 0 (already there)
E[i] = 1 + (1/6) × Σ E[i + d] for d = 1..6 (if i+d ≤ N)
Solve from E[N] backwards to E[0]. 
# Expected steps on a 1D random walk
def expected_steps(n):
    # dp[i] = expected steps from position i to reach n
    dp = [0.0] * (n + 1)
    for i in range(n - 1, -1, -1):
        transitions = [j for j in range(i+1, min(i+7, n+1))]
        dp[i] = 1 + sum(dp[j] for j in transitions) / len(transitions)
    return dp[0]

Advanced Techniques

CF 1800+ territory — topics you won't have seen at A-Level.

● Hard (CF 1600–2000)

Matrix Exponentiation Hard

If a recurrence can be written as a matrix multiplication, fast exponentiation computes the n-th term in O(k³ log n) where k = matrix size.

Fibonacci via Matrix — O(log n) [F(n+1)] [1 1]ⁿ [F(1)]
[F(n) ] = [1 0] × [F(0)]
Any linear recurrence with constant coefficients can be expressed this way. Very powerful for DP optimisation.

Extended Euclidean Algorithm Hard

Bézout's Identity For any integers a, b: there exist x, y such that
ax + by = gcd(a, b)
Extended GCD finds x, y. Used for: modular inverse (when mod not prime), linear Diophantine equations.
# Extended GCD — returns (gcd, x, y) where a*x + b*y = gcd
def extgcd(a, b):
    if b == 0: return a, 1, 0
    g, x, y = extgcd(b, a % b)
    return g, y, x - (a // b) * y

# Modular inverse using extgcd (works for non-prime mod too)
def mod_inv(a, m):
    g, x, _ = extgcd(a, m)
    if g != 1: return -1  # no inverse if gcd != 1
    return x % m
● Expert (CF 2000+)

Game Theory — Sprague-Grundy Theorem Expert

For combinatorial games (two players, perfect information, no randomness, last move wins). Every position has a Grundy number (nimber). A position is losing (for the player to move) iff its Grundy number is 0.

Grundy Number (mex = minimum excludant) G(position) = mex({ G(reachable positions) })
mex({0,1,3}) = 2 (smallest non-negative integer not in set)
Combined game XOR rule: XOR all Grundy numbers. If XOR = 0 → current player LOSES (second player wins).
Nim Game — one pile of n stones, remove 1-3 per turn
G(0)=0

G(4)=0

G(n) = n mod 4. Positions with G=0 are P-positions (previous player wins = current player loses).

Chinese Remainder Theorem Expert

CRT — System of Congruences x ≡ a₁ (mod m₁)
x ≡ a₂ (mod m₂) → unique x mod (m₁ × m₂) if gcd(m₁,m₂)=1
x ≡ a₃ (mod m₃)
Find x using: x = a₁M₁y₁ + a₂M₂y₂ + ... where Mᵢ = M/mᵢ and yᵢ = Mᵢ⁻¹ mod mᵢ

Fast Fourier Transform (FFT) Expert

Multiplies two polynomials of degree n in O(n log n) instead of O(n²). This is how you convolve arrays — like counting pairs with sum k across two arrays.

When to suspect FFT: "Count pairs (i,j) such that a[i] + b[j] = k." Convert arrays to polynomials, multiply, coefficient of xᵏ in the product = answer.
